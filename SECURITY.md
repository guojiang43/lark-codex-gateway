# Security Policy

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability, exposed credential, or privacy leak.

Use the repository's **Security** tab to open a private GitHub Security Advisory. Include the affected version, reproduction steps, impact, and any proposed mitigation. Remove real App Secrets, tokens, private keys, user messages, local paths, and Codex authentication data from the report.

## Supported version

Security fixes target the latest commit on `main`. This project is pre-1.0; deployment-specific integrations should be revalidated after every upgrade.
