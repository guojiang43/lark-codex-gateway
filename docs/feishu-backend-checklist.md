# 飞书后台配置清单

这一步必须在飞书开放平台的机器人应用后台完成并发布应用版本。代码中的事件 key 已固定；后台必须逐字一致。

## 1. 事件与回调

1. 在“事件配置”中使用长连接接收事件。
2. 保留已有的 `im.message.receive_v1`（接收消息 v2.0）。
3. 新增 `application.bot.menu_v6`（机器人自定义菜单）。
4. 在“回调配置”中选择长连接，并新增 `card.action.trigger`（卡片回传交互）。不要配置旧版卡片回调。

## 2. 机器人自定义菜单

开启机器人自定义菜单，选择悬浮菜单或可切换菜单均可。三个主菜单及子菜单如下；每个子菜单的响应动作都选择“推送事件”。日常界面不区分工程/分析任务，也不暴露分叉或只读模式。

| 主菜单 | 子菜单 | 事件 key |
| --- | --- | --- |
| 会话 | 新建会话 | `new_session` |
| 会话 | 切换会话 | `session_select` |
| 会话 | 当前会话 | `session_current` |
| 管理 | 重命名会话 | `session_rename` |
| 管理 | 归档会话 | `session_archive` |
| 管理 | 运行状态 | `current_status` |
| 控制 | 停止生成 | `stop_generation` |
| 控制 | 使用帮助 | `help` |

## 3. 发布前核对

- 菜单动作不是“发送文字消息”，而是“推送事件”。
- 卡片交互使用新版 `card.action.trigger` 长连接回调。
- 应用可用范围包含目标使用者。
- 保存后创建并发布一个应用版本；未发布的菜单配置不会在飞书客户端生效。

发布完成后，在机器人单聊依次验证：

1. 点击“会话 → 切换会话”，应收到 Card 2.0 Session 列表。
2. 点击“会话 → 新建会话”，应创建并切换到空白 Session。
3. 在 Session 卡片中执行切换、重命名和归档。
4. 发起一个较长任务后点击“控制 → 停止生成”，应中断当前 turn。
